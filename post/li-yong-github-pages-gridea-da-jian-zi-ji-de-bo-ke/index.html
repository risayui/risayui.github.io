利用 Github Pages + Gridea 搭建自己的博客

2022-05-28 3 min read

为什么要搭建自己的博客

  • 锻炼自己表达、输出的能力。高考后便很少书写大段的文字，以至于现在语言匮乏、难以流畅地表达自己的观点、缺乏逻辑性。希望通过写博客复建、培养自己讲好故事的能力。
  • 记录自己的成长。
  • 培养动手能力。虽然国内已有不少知识分享的平台，但作为一个技术爱好者，还是很希望能部署自己的网页（哪怕是最简单的方式），在搭建和写文章的过程中也可以帮助自己熟悉 Git 操作与 Markdown 语法。

介绍

  • Github Pages：GitHub 提供的免费服务，可以为 Github 仓库中的静态网页生成访问域名。https://pages.github.com/
  • Gridea：一个静态博客写作客户端，帮助你更容易地构建并管理博客或任何静态站点。 https://gridea.dev/
  • 静态网站：静态网站是指全部由 HTML 页面组成的网站，所有的内容包含在网页文件中。不需要通过数据库工作。

创建托管仓库

仓库名设为 username.github.io

创建后，点击仓库的 Settings-Pages
点击 choose a theme 设置好主题后，便可以通过网址访问博客了

申请Personal access token

点击头像-settings-developer settings-Personal access tokens-Generate new token
Note：随便写，说明 token 是来干什么用的。
Expiration：token 的到期时间，这里我嫌麻烦就选择永不过期
token 的权限只选择 repo

保存好生成的 token，它不会再显示第二次

添加 Gittalk App

添加 Gittalk 插件实现评论功能
点击头像-settings-developer settings-Oauth Apps-Register a new application

Application name 随意填写，两个 URL 都填写自己的博客地址（username.github.io）

生成 Client ID 和 Client secrets，注意保存好

配置Gridea

基础配置，点击[检测远程连接]，确认连接成功后，保存。

评论插件配置，填入刚才生成的 Client ID 和 Client secrets

完成配置后，点击同步，将代码 push 到 Github 仓库
刷新网址，可以看到网页配置已成功生效。

书写第一篇文章

完善 Gridea 默认生成的 about 文章
点击保存，可以预览发布效果

最后点击同步，可以看到文章已成功发布👏
